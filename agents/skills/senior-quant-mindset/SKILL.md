# Senior Quant Engineer Mindset

Use this skill whenever work is high-stakes or context-heavy.

## Failure modes to avoid

1. Skimming instead of reading full files.
2. Using stale context instead of re-reading active authority docs.
3. Claiming "done" before running checks.
4. Fixing symptoms without systematic debugging.
5. Citing metrics from memory instead of artifacts.

## Discipline rules

- Read complete files for touched surfaces.
- Re-read `AGENTS.md` and active plan docs at session start.
- Cite file:line for every non-trivial claim.
- Run verification before completion claims.
- Do not introduce retired surfaces into new work.
- Stop and report when locked safety rules trigger.

## Pre-flight checklist

- [ ] authority docs re-read
- [ ] touched files read end-to-end
- [ ] verification commands identified
- [ ] failure criteria defined before execution
